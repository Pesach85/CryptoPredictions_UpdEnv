1. ASSET-SPECIFIC BEHAVIOR:
   - ETH, ADA, BCH: Focused+lags=14 improves directional accuracy (+1.0pp to +2.8pp)
   - XBT, LTC: Baseline lags=30 clearly better (needs longer context)
   - SOL: Focused improves MAPE significantly (-0.60pp), but directional accuracy drops

2. MAPE EFFECT:
   - Focused+lags=14 does NOT improve overall MAPE (stays at 3.21%)
   - However, Focused DOES improve SOL MAPE from 4.81% → 4.21% (now passes strict 4.0% gate!)

3. DIRECTIONAL ACCURACY TRADE-OFF:
   - Lags=14 reduces dimensionality but may discard long-range patterns
   - BTC, LTC need 30+ lags to capture sustained trends
   - Newer/volatile assets (ADA, SOL, ETH) benefit from noise reduction at lags=14

4. RECOMMENDED STRATEGY:
   - Use lags=30 for established, trending assets (XBT, LTC) — better magnitude accuracy
   - Use lags=14 for volatile/newer assets (ADA, SOL, ETH) — better directional accuracy + reduced noise
   - Use focused mode for SOL specifically — improves MAPE below 4.21% gate threshold

5. NO UNIVERSAL WINNER:
   - Close+lags=30 wins on magnitude error (MAPE)
   - Focused+lags=14 wins only on specific assets (ADA directional, SOL MAPE)
   - "One size fits all" is suboptimal; hybrid approach per asset is superior
